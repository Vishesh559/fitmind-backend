# FitMind Backend

FastAPI backend for FitMind AI — handles authentication, workout/nutrition tracking, and AI-powered coaching.

## 🏗️ Architecture

```
[Next.js Frontend] --> [FastAPI Backend] --> [PostgreSQL Database]
                              |
                       [Anthropic API]
```

Built with FastAPI, SQLAlchemy ORM, and PostgreSQL. AI coaching features are powered by the Anthropic API.

## ✨ Features

- **Auth** — JWT-based authentication (`python-jose`) with password hashing via `passlib`
- **Workouts** — CRUD endpoints for logging and retrieving workout data
- **Nutrition** — CRUD endpoints for logging and retrieving nutrition data
- **AI Coaching** — endpoints that use the Anthropic API to generate personalized fitness/nutrition advice based on a user's logged data
- CORS configured for local frontend development (`localhost:3000`)

## 🛠️ Tech Stack

| Component | Technology |
|---|---|
| Framework | FastAPI |
| ORM | SQLAlchemy |
| Database | PostgreSQL (via `psycopg2`) |
| Auth | JWT (`python-jose`) + `passlib` for hashing |
| AI | Anthropic API |
| Server | Uvicorn |

## 🚀 Getting Started

### Prerequisites
- Python 3.10+
- PostgreSQL instance running locally or remotely
- An Anthropic API key

### Installation

```bash
git clone https://github.com/Vishesh559/fitmind-backend.git
cd fitmind-backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Environment Variables

Create a `.env` file in the project root with:

```
DATABASE_URL=postgresql://<user>:<password>@<host>:<port>/<dbname>
ANTHROPIC_API_KEY=<your-anthropic-api-key>
SECRET_KEY=<your-jwt-secret-key>
```

### Running locally

```bash
uvicorn main:app --reload
```

The API will be available at `http://localhost:8000`. Interactive docs are auto-generated by FastAPI at `http://localhost:8000/docs`.

## 📡 API Endpoints

| Prefix | Purpose |
|---|---|
| `/auth` | Registration, login, token handling |
| `/workouts` | Create/read/update/delete workout logs |
| `/nutrition` | Create/read/update/delete nutrition logs |
| `/ai` | AI-generated coaching insights |

Full request/response schemas are visible in the auto-generated docs at `/docs` once the server is running.
